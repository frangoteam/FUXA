## 字节序详解

### 大端字节序 (Big Endian / BE)
- **存储方式**: 高位字节存储在低地址
- **示例**: 0x1234 存储为 [0x12, 0x34]
- **应用场景**: 网络协议、Modbus标准

### 小端字节序 (Little Endian / LE)
- **存储方式**: 低位字节存储在低地址
- **示例**: 0x1234 存储为 [0x34, 0x12]
- **应用场景**: x86架构处理器

### Modbus中间序 (Mid-Little Endian / MLE)
- **存储方式**: 寄存器内部使用大端，寄存器之间使用小端
- **示例**: 32位值 0x12345678
  - 分为两个寄存器: 0x1234 和 0x5678
  - 存储顺序: [0x56, 0x78, 0x12, 0x34]
- **应用场景**: 某些PLC设备

### 推荐使用数据类型
所有测试的数据类型都可以在 FUXA 中安全使用：

**整数类型**:
- Int16 / Int16LE (16位有符号整数)
- Int32 / Int32MLE (32位有符号整数)
- UInt32 / UInt32LE / UInt32MLE (32位无符号整数)
- Int64 / Int64LE (64位有符号整数)

**浮点类型**:
- Float32 / Float32LE / Float32MLE (32位浮点数)
- Float64 / Float64LE / Float64MLE (64位浮点数)

### 注意事项
1. Float32 类型存在微小精度损失（约0.00001），这是IEEE 754标准的正常现象
2. Int64 类型使用 BigInt 处理，确保数值精度
3. 不同字节序适用于不同厂商的PLC设备，请根据实际设备选择
